15    /**
16    * As Heston like stochastic volatility model, using a process \( \lambda(t) = \sqrt(V(t)) \)
17    * \[
18    * dV(t) = \kappa ( \theta - V(t) ) dt + \xi \sqrt{V(t)} dW_{1}(t), \quad V(0) = 1.0,
19    * \]
20    * where \( \lambda(0) = 1 \) to scale all factor loadings \( f_{i} \) returned by a given covariance model.
21    *
22    * The model constructed is \( \lambda(t) F(t) \) where \( \lambda(t) \) is
23    * a discretization of the above process and \( F = ( f_{1}, \ldots, f_{m} ) \) is the factor loading
24    * from the given covariance model.
25    *
26    * The process uses the first factor of the Brownian motion provided by an object implementing
27    * {@link net.finmath.montecarlo.BrownianMotion}. This can be used to generate correlations to
28    * other objects. If you like to reuse a factor of another Brownian motion use a
29    * {@link net.finmath.montecarlo.BrownianMotionView}
30    * to delegate \( ( \mathrm{d} W_{1}(t) ) \) to a different object.
31    *
32    * The parameter of this model is a joint parameter vector, consisting
33    * of the parameter vector of the given base covariance model and
34    * appending the parameters <i>&kappa;</i>, <i>&theta;</i> and <i>&xi;</i> at the end.
35    *
36    * If this model is not calibrateable, its parameter vector is that of the
37    * covariance model, i.e., <i>&nu;</i> and <i>&rho;</i> will be not
38    * part of the calibration.
39    *
40    * For an illustration of its usage see the associated unit test.
41    *
42    * @author Christian Fries
43    * @version 1.0
44    */
