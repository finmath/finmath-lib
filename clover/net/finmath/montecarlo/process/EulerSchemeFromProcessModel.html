21    /**
22    * This class implements some numerical schemes for multi-dimensional multi-factor Ito process.
23    *
24    * <p>
25    * It features the standard Euler scheme and the standard predictor-corrector Euler scheme
26    * for <i>Y</i>, then applies the <i>state space transform</i> \( X = f(Y) \).
27    * </p>
28    *
29    * <p>
30    * For the standard Euler scheme the process \( Y = (Y_{1},\ldots,Y_{d}) \) is discretized as
31    * \[
32    * Y_{j}(t_{i+1}) = Y_{j}(t_{i}) + \mu_{j}(t_{i}) \Delta t_{i} + \lambda_{1,j}(t_{i}) \Delta W_{1}(t_{i}) + \ldots + \lambda_{m,j} \Delta W_{m} \text{.}
33    * \]
34    * </p>
35    *
36    * The parameters have to be provided by a {@link net.finmath.montecarlo.model.ProcessModel}:
37    * <ul>
38    * <li> \( f \) - <code>applyStateSpaceTransform</code> </li>
39    * <li> \( Y(t_{0}) \) - <code>getInitialState</code> </li>
40    * <li> \( \mu \) - <code>getDrift</code> </li>
41    * <li> \( \lambda_{j} \) - <code>getFactorLoading</code> </li>
42    * </ul>
43    *
44    * <p>
45    * Using the <i>state space transform</i> \( (t,x) \mapsto \exp(x) \), it is possible to create a log-Euler scheme, i.e.,
46    * \[
47    * X(t_{i+1}) = X(t_{i}) \cdot \exp\left( (r(t_{i}) - \frac{1}{2} \sigma(t_{i})^2) \Delta t_{i} + \sigma(t_{i}) \Delta W(t_{i}) \right) \text{.}
48    * \]
49    * for a process \( \mathrm{d} X = r X \mathrm{d}t + \sigma X \mathrm{d}W \).
50    * </p>
51    *
52    * The dimension \( d \) is called <code>numberOfComponents</code> here.
53    * The value \( m \) is called <code>numberOfFactors</code> here.
54    * The default for <code>numberOfFactors</code> is 1.
55    *
56    * @author Christian Fries
57    * @see net.finmath.montecarlo.model.ProcessModel The interface definition contains more details.
58    * @version 1.4
59    */
