25    /**
26    * This class implements a multi-asset Black Schole Model as Monte-Carlo simulation implementing <code>AssetModelMonteCarloSimulationModel</code>.
27    *
28    * The model is
29    * \[
30    * dS_{i} = r S_{i} dt + \sigma_{i} S_{i} dW_{i}, \quad S_{i}(0) = S_{i,0},
31    * \]
32    * \[
33    * dN = r N dt, \quad N(0) = N_{0},
34    * \]
35    * \[
36    * dW_{i} dW_{j} = \rho_{i,j} dt,
37    * \]
38    *
39    * The class provides the model of \( S_{i} \) to an <code>{@link net.finmath.montecarlo.process.MonteCarloProcess}</code> via the specification of
40    * \( f = exp \), \( \mu_{i} = r - \frac{1}{2} \sigma_{i}^2 \), \( \lambda_{i,j} = \sigma_{i} g_{i,j} \), i.e.,
41    * of the SDE
42    * \[
43    * dX_{i} = \mu_{i} dt + \lambda_{i,j} dW, \quad X_{i}(0) = \log(S_{i,0}),
44    * \]
45    * with \( S = f(X) \). See {@link net.finmath.montecarlo.process.MonteCarloProcess} for the notation.
46    *
47    * @author Christian Fries
48    * @author Roland Bachl
49    * @see net.finmath.montecarlo.process.MonteCarloProcess The interface for numerical schemes.
50    * @see net.finmath.montecarlo.model.ProcessModel The interface for models provinding parameters to numerical schemes.
51    * @version 1.1
52    */
