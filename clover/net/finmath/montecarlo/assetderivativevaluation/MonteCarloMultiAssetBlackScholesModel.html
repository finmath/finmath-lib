25    /**
26    * This class glues together a <code>BlackScholeModel</code> and a Monte-Carlo implementation of a <code>MonteCarloProcessFromProcessModel</code>
27    * and forms a Monte-Carlo implementation of the Black-Scholes Model by implementing <code>AssetModelMonteCarloSimulationModel</code>.
28    *
29    * The model is
30    * \[
31    * dS_{i} = r S_{i} dt + \sigma_{i} S_{i} dW_{i}, \quad S_{i}(0) = S_{i,0},
32    * \]
33    * \[
34    * dN = r N dt, \quad N(0) = N_{0},
35    * \]
36    * \[
37    * dW_{i} dW_{j} = \rho_{i,j} dt,
38    * \]
39    *
40    * The class provides the model of \( S_{i} \) to an <code>{@link net.finmath.montecarlo.process.MonteCarloProcess}</code> via the specification of
41    * \( f = exp \), \( \mu_{i} = r - \frac{1}{2} \sigma_{i}^2 \), \( \lambda_{i,j} = \sigma_{i} g_{i,j} \), i.e.,
42    * of the SDE
43    * \[
44    * dX_{i} = \mu_{i} dt + \lambda_{i,j} dW, \quad X_{i}(0) = \log(S_{i,0}),
45    * \]
46    * with \( S = f(X) \). See {@link net.finmath.montecarlo.process.MonteCarloProcess} for the notation.
47    *
48    * @author Christian Fries
49    * @see net.finmath.montecarlo.process.MonteCarloProcess The interface for numerical schemes.
50    * @see net.finmath.montecarlo.model.ProcessModel The interface for models provinding parameters to numerical schemes.
51    * @version 1.0
52    */
