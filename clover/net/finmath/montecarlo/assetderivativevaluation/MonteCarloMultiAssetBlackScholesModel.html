21    /**
22    * This class glues together a <code>BlackScholeModel</code> and a Monte-Carlo implementation of a <code>MonteCarloProcessFromProcessModel</code>
23    * and forms a Monte-Carlo implementation of the Black-Scholes Model by implementing <code>AssetModelMonteCarloSimulationModel</code>.
24    *
25    * The model is
26    * \[
27    * dS_{i} = r S_{i} dt + \sigma_{i} S_{i} dW_{i}, \quad S_{i}(0) = S_{i,0},
28    * \]
29    * \[
30    * dN = r N dt, \quad N(0) = N_{0},
31    * \]
32    * \[
33    * dW_{i} dW_{j} = \rho_{i,j} dt,
34    * \]
35    *
36    * The class provides the model of \( S_{i} \) to an <code>{@link net.finmath.montecarlo.process.MonteCarloProcess}</code> via the specification of
37    * \( f = exp \), \( \mu_{i} = r - \frac{1}{2} \sigma_{i}^2 \), \( \lambda_{i,j} = \sigma_{i} g_{i,j} \), i.e.,
38    * of the SDE
39    * \[
40    * dX_{i} = \mu_{i} dt + \lambda_{i,j} dW, \quad X_{i}(0) = \log(S_{i,0}),
41    * \]
42    * with \( S = f(X) \). See {@link net.finmath.montecarlo.process.MonteCarloProcess} for the notation.
43    *
44    * @author Christian Fries
45    * @see net.finmath.montecarlo.process.MonteCarloProcess The interface for numerical schemes.
46    * @see net.finmath.montecarlo.model.ProcessModel The interface for models provinding parameters to numerical schemes.
47    * @version 1.0
48    */
