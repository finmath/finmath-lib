23    /**
24    * This class implements a parallel Levenberg-Marquardt non-linear least-squares fit
25    * algorithm.
26    *
27    * <p>
28    * The solver minimizes \( || f ||_{L_{2}} \) for a function \( f:\mathbb{R}^n \rightarrow \mathbb{R}^m \).
29    * The solver requires the calculation of a Jacobi-matrix \( J = \frac{\mathrm{d}f}{\mathrm{d}x} \). The iteration steps
30    * are then defined by
31    * \[
32    * \Delta x = H_{\lambda}^{-1} J^T f
33    * \]
34    * where \( H_{\lambda} \) is a regularized approximation of the Hessian matrix.
35    * The solver supports two different regularizations. For <code>RegularizationMethod.LEVENBERG</code> the solver uses
36    * \( H_{\lambda} = J^T J + \lambda I \). For <code>RegularizationMethod.LEVENBERG_MARQUARDT</code> the solver uses
37    * \( H_{\lambda} = J^T J + \lambda \text{diag}(J^T J) \).
38    * </p>
39    *
40    * <p>
41    * The design avoids the need to define the objective function as a
42    * separate class. The objective function is defined by overriding a class
43    * method, see the sample code below.
44    * </p>
45   
46    * <p>
47    * The Levenberg-Marquardt solver is implemented in using multi-threading.
48    * The calculation of the derivatives (in case a specific implementation of
49    * {@code setDerivatives(double[] parameters, double[][] derivatives)} is not
50    * provided) may be performed in parallel by setting the parameter <code>numberOfThreads</code>.
51    * </p>
52    *
53    * <p>
54    * To use the solver inherit from it and implement the objective function as
55    * {@code setValues(double[] parameters, double[] values)} where values has
56    * to be set to the value of the objective functions for the given parameters.
57    * <br>
58    * You may also provide an a derivative for your objective function by
59    * additionally overriding the function {@code setDerivatives(double[] parameters, double[][] derivatives)},
60    * otherwise the solver will calculate the derivative via finite differences.
61    * </p>
62    * <p>
63    * To reject a point, it is allowed to set an element of <code>values</code> to {@link java.lang.Double#NaN}
64    * in the implementation of {@code setValues(double[] parameters, double[] values)}.
65    * Put differently: The solver handles NaN values in <code>values</code> as an error larger than
66    * the current one (regardless of the current error) and rejects the point.
67    * <br>
68    * Note, however, that is is an error if the initial parameter guess results in an NaN value.
69    * That is, the solver should be initialized with an initial parameter in an admissible region.
70    * </p>
71    *
72    * The following simple example finds a solution for the equation <br>
73    * <table>
74    * <caption>Sample linear system of equations.</caption>
75    * <tr><td>
76    * 0.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 5.0
77    * </td></tr>
78    * <tr><td>
79    * 2.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 10.0
80    * </td></tr>
81    * </table>
82    *
83    * <pre>
84    * <code>
85    * LevenbergMarquardt optimizer = new LevenbergMarquardt() {
86    * // Override your objective function here
87    * public void setValues(double[] parameters, double[] values) {
88    * values[0] = parameters[0] * 0.0 + parameters[1];
89    * values[1] = parameters[0] * 2.0 + parameters[1];
90    * }
91    * };
92    *
93    * // Set solver parameters
94    * optimizer.setInitialParameters(new double[] { 0, 0 });
95    * optimizer.setWeights(new double[] { 1, 1 });
96    * optimizer.setMaxIteration(100);
97    * optimizer.setTargetValues(new double[] { 5, 10 });
98    *
99    * optimizer.run();
100    *
101    * double[] bestParameters = optimizer.getBestFitParameters();
102    * </code>
103    * </pre>
104    *
105    * See the example in the main method below.
106    *
107    * <p>
108    * The class can be initialized to use a multi-threaded valuation. If initialized
109    * this way the implementation of <code>setValues</code> must be thread-safe.
110    * The solver will evaluate the gradient of the value vector in parallel, i.e.,
111    * use as many threads as the number of parameters.
112    * </p>
113    *
114    * Note: Iteration steps will be logged (java.util.logging) with LogLevel.FINE
115    *
116    * @author Christian Fries
117    * @version 1.6
118    */
