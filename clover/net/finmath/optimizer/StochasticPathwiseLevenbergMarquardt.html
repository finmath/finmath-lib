26    /**
27    * This class implements a stochastic Levenberg Marquardt non-linear least-squares fit
28    * algorithm.
29    * <p>
30    * The design avoids the need to define the objective function as a
31    * separate class. The objective function is defined by overriding a class
32    * method, see the sample code below.
33    * </p>
34    *
35    * <p>
36    * The Levenberg-Marquardt solver is implemented in using multi-threading.
37    * The calculation of the derivatives (in case a specific implementation of
38    * {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)} is not
39    * provided) may be performed in parallel by setting the parameter <code>numberOfThreads</code>.
40    * </p>
41    *
42    * <p>
43    * To use the solver inherit from it and implement the objective function as
44    * {@code setValues(RandomVariable[] parameters, RandomVariable[] values)} where values has
45    * to be set to the value of the objective functions for the given parameters.
46    * <br>
47    * You may also provide an a derivative for your objective function by
48    * additionally overriding the function {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)},
49    * otherwise the solver will calculate the derivative via finite differences.
50    * </p>
51    * <p>
52    * To reject a point, it is allowed to set an element of <code>values</code> to {@link java.lang.Double#NaN}
53    * in the implementation of {@code setValues(RandomVariable[] parameters, RandomVariable[] values)}.
54    * Put differently: The solver handles NaN values in <code>values</code> as an error larger than
55    * the current one (regardless of the current error) and rejects the point.
56    * <br>
57    * Note, however, that is is an error if the initial parameter guess results in an NaN value.
58    * That is, the solver should be initialized with an initial parameter in an admissible region.
59    * </p>
60    *
61    * The following simple example finds a solution for the equation <br>
62    * <table>
63    * <caption>Sample linear system of equations.</caption>
64    * <tr><td>
65    * 0.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 5.0
66    * </td></tr>
67    * <tr><td>
68    * 2.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 10.0
69    * </td></tr>
70    * </table>
71    *
72    * <pre>
73    * <code>
74    * LevenbergMarquardt optimizer = new LevenbergMarquardt() {
75    * // Override your objective function here
76    * public void setValues(RandomVariable[] parameters, RandomVariable[] values) {
77    * values[0] = parameters[0] * 0.0 + parameters[1];
78    * values[1] = parameters[0] * 2.0 + parameters[1];
79    * }
80    * };
81    *
82    * // Set solver parameters
83    * optimizer.setInitialParameters(new RandomVariable[] { 0, 0 });
84    * optimizer.setWeights(new RandomVariable[] { 1, 1 });
85    * optimizer.setMaxIteration(100);
86    * optimizer.setTargetValues(new RandomVariable[] { 5, 10 });
87    *
88    * optimizer.run();
89    *
90    * RandomVariable[] bestParameters = optimizer.getBestFitParameters();
91    * </code>
92    * </pre>
93    *
94    * See the example in the main method below.
95    *
96    * <p>
97    * The class can be initialized to use a multi-threaded valuation. If initialized
98    * this way the implementation of <code>setValues</code> must be thread-safe.
99    * The solver will evaluate the gradient of the value vector in parallel, i.e.,
100    * use as many threads as the number of parameters.
101    * </p>
102    *
103    * Note: Iteration steps will be logged (java.util.logging) with LogLevel.FINE
104    *
105    * @author Christian Fries
106    * @version 1.6
107    */
