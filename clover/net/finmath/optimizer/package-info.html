1    /**
2    <p>
3    This package provides classes with numerical algorithm for optimization of
4    an objective function and a factory to easy construction of the optimizers.
5    </p>
6   
7    <b>Why a package for optimization algorithms?</b>
8    <p>
9    Given that there are a variety of numerical libraries featuring optimization algorithms
10    (e.g., Apache Commons Math), why do we provide a package inside finmath lib?
11    This packages provides a unified interface for passing optimizers to other classes
12    via an <code>OptimizationFactoryInterface</code> and an <code>Optimizer</code>
13    and an <code>Optimizer.ObjectiveFunction</code>.
14    This allows use of different optimization frameworks without bothering with the
15    framework specific constructors and framework specific definitions of objective functions.
16    </p>
17   
18    <p>
19    A class implementing the <code>OptimizationFactoryInterface</code> allows the
20    specification of parameters specific to the optimizer, but leave the specification
21    of the initial values and the objective function still open. It provides a factory
22    method which takes the objective function and initial values as parameters and
23    constructs the specific optimizer by returning an object implementing
24    <code>Optimizer</code>.
25    </p>
26   
27    <b>Example</b>
28   
29    <p>
30   
31    The following code is an example of an optimization problem using an <code>OptimizerFactory</code>
32    as argument.
33    <pre><code>
34    public void testOptimizerWithRosenbrockFunction(OptimizerFactory optimizerFactory) throws SolverException {
35    Optimizer.ObjectiveFunction objectiveFunction = new Optimizer.ObjectiveFunction() {
36    public void setValues(double[] parameters, double[] values) {
37    values[0] = 10.0 * (parameters[1] - parameters[0]*parameters[0]);
38    values[1] = 1.0 - parameters[0];
39    }
40    };
41   
42    Optimizer optimizer = optimizerFactory.getOptimizer(
43    objectiveFunction,
44    new double[] { 0.5, 0.5 }, // initialParameters
45    new double[] { Double.NEGATIVE_INFINITY, Double.NEGATIVE_INFINITY }, // lowerBound
46    new double[] { Double.POSITIVE_INFINITY, Double.POSITIVE_INFINITY }, // upperBound
47    new double[] { 0.5, 0.5 }, // parameterStep
48    new double[] { 0.0, 0.0 }); // targetValues
49   
50    optimizer.run();
51   
52    double[] bestParameters = optimizer.getBestFitParameters();
53    System.out.println("The solver " + optimizer.getClass() + " for problem 'Rosebrock' required " + optimizer.getIterations() + " iterations. Accuracy is " + optimizer.getRootMeanSquaredError() + ". The best fit parameters are:");
54    for (int i = 0; i &lt; bestParameters.length; i++) System.out.println("\tparameter[" + i + "]: " + bestParameters[i]);
55   
56    System.out.println();
57   
58    Assert.assertTrue(Math.abs(bestParameters[0] - 1.0) &lt; 1E-10);
59    Assert.assertTrue(Math.abs(bestParameters[1] - 1.0) &lt; 1E-10);
60    }
61    </code></pre>
62   
63    Now, we may pass different optimizers to the optimization problem. For example the CMA-ES solver from commons math:
64    <pre>
65    public void testRosenbrockFunctionWithCMAES() throws SolverException {
66   
67    OptimizerFactory optimizerFactory = new OptimizerFactoryCMAES(0.0, 200);
68    this.testOptimizerWithRosenbrockFunction(optimizerFactory);
69    }
70    </pre>
71   
72    Or the multi-threadded Levenberg Marquardt solver (using two threads) from finmath-lib:
73    <pre>
74    public void testRosenbrockFunctionWithLevenbergMarquard() throws SolverException {
75   
76    OptimizerFactory optimizerFactory = new OptimizerFactoryLevenbergMarquardt(200, 2);
77    this.testOptimizerWithRosenbrockFunction(optimizerFactory);
78    }
79    </pre>
80   
81    <b>Optimization algorithms</b>
82    <p>
83    The package also contains an implementation of the Levenberg Marquardt optimizer,
84    a multi-dimensional non-linear least-square.
85    In addition we provide wrappers (via specific <code>OptimizationFactoryInterface</code>
86    implementations) to some optimizers from Apache commons-math.
87    </p>
88    */
