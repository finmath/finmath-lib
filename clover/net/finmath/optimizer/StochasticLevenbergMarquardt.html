25    /**
26    * This class implements a stochastic Levenberg Marquardt non-linear least-squares fit
27    * algorithm.
28    * <p>
29    * The design avoids the need to define the objective function as a
30    * separate class. The objective function is defined by overriding a class
31    * method, see the sample code below.
32    * </p>
33    *
34    * <p>
35    * The Levenberg-Marquardt solver is implemented in using multi-threading.
36    * The calculation of the derivatives (in case a specific implementation of
37    * {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)} is not
38    * provided) may be performed in parallel by setting the parameter <code>numberOfThreads</code>.
39    * </p>
40    *
41    * <p>
42    * To use the solver inherit from it and implement the objective function as
43    * {@code setValues(RandomVariable[] parameters, RandomVariable[] values)} where values has
44    * to be set to the value of the objective functions for the given parameters.
45    * <br>
46    * You may also provide an a derivative for your objective function by
47    * additionally overriding the function {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)},
48    * otherwise the solver will calculate the derivative via finite differences.
49    * </p>
50    * <p>
51    * To reject a point, it is allowed to set an element of <code>values</code> to {@link java.lang.Double#NaN}
52    * in the implementation of {@code setValues(RandomVariable[] parameters, RandomVariable[] values)}.
53    * Put differently: The solver handles NaN values in <code>values</code> as an error larger than
54    * the current one (regardless of the current error) and rejects the point.
55    * <br>
56    * Note, however, that is is an error if the initial parameter guess results in an NaN value.
57    * That is, the solver should be initialized with an initial parameter in an admissible region.
58    * </p>
59    *
60    * The following simple example finds a solution for the equation <br>
61    * <table>
62    * <caption>Sample linear system of equations.</caption>
63    * <tr><td>
64    * 0.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 5.0
65    * </td></tr>
66    * <tr><td>
67    * 2.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 10.0
68    * </td></tr>
69    * </table>
70    *
71    * <pre>
72    * <code>
73    * LevenbergMarquardt optimizer = new LevenbergMarquardt() {
74    * // Override your objective function here
75    * public void setValues(RandomVariable[] parameters, RandomVariable[] values) {
76    * values[0] = parameters[0] * 0.0 + parameters[1];
77    * values[1] = parameters[0] * 2.0 + parameters[1];
78    * }
79    * };
80    *
81    * // Set solver parameters
82    * optimizer.setInitialParameters(new RandomVariable[] { 0, 0 });
83    * optimizer.setWeights(new RandomVariable[] { 1, 1 });
84    * optimizer.setMaxIteration(100);
85    * optimizer.setTargetValues(new RandomVariable[] { 5, 10 });
86    *
87    * optimizer.run();
88    *
89    * RandomVariable[] bestParameters = optimizer.getBestFitParameters();
90    * </code>
91    * </pre>
92    *
93    * See the example in the main method below.
94    *
95    * <p>
96    * The class can be initialized to use a multi-threaded valuation. If initialized
97    * this way the implementation of <code>setValues</code> must be thread-safe.
98    * The solver will evaluate the gradient of the value vector in parallel, i.e.,
99    * use as many threads as the number of parameters.
100    * </p>
101    *
102    * Note: Iteration steps will be logged (java.util.logging) with LogLevel.FINE
103    *
104    * @author Christian Fries
105    * @version 1.6
106    */
