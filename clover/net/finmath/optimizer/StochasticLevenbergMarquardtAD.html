19    /**
20    * This class implements a stochastic Levenberg Marquardt non-linear least-squares fit
21    * algorithm.
22    * <p>
23    * The design avoids the need to define the objective function as a
24    * separate class. The objective function is defined by overriding a class
25    * method, see the sample code below.
26    * </p>
27    *
28    * <p>
29    * The Levenberg-Marquardt solver is implemented in using multi-threading.
30    * The calculation of the derivatives (in case a specific implementation of
31    * {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)} is not
32    * provided) may be performed in parallel by setting the parameter <code>numberOfThreads</code>.
33    * </p>
34    *
35    * <p>
36    * To use the solver inherit from it and implement the objective function as
37    * {@code setValues(RandomVariable[] parameters, RandomVariable[] values)} where values has
38    * to be set to the value of the objective functions for the given parameters.
39    * <br>
40    * You may also provide an a derivative for your objective function by
41    * additionally overriding the function {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)},
42    * otherwise the solver will calculate the derivative via finite differences.
43    * </p>
44    * <p>
45    * To reject a point, it is allowed to set an element of <code>values</code> to {@link java.lang.Double#NaN}
46    * in the implementation of {@code setValues(RandomVariable[] parameters, RandomVariable[] values)}.
47    * Put differently: The solver handles NaN values in <code>values</code> as an error larger than
48    * the current one (regardless of the current error) and rejects the point.
49    * <br>
50    * Note, however, that is is an error if the initial parameter guess results in an NaN value.
51    * That is, the solver should be initialized with an initial parameter in an admissible region.
52    * </p>
53    *
54    * The following simple example finds a solution for the equation <br>
55    * <table>
56    * <caption>Sample linear system of equations.</caption>
57    * <tr><td>
58    * 0.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 5.0
59    * </td></tr>
60    * <tr><td>
61    * 2.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 10.0
62    * </td></tr>
63    * </table>
64    *
65    * <pre>
66    * <code>
67    * LevenbergMarquardt optimizer = new LevenbergMarquardt() {
68    * // Override your objective function here
69    * public void setValues(RandomVariable[] parameters, RandomVariable[] values) {
70    * values[0] = parameters[0] * 0.0 + parameters[1];
71    * values[1] = parameters[0] * 2.0 + parameters[1];
72    * }
73    * };
74    *
75    * // Set solver parameters
76    * optimizer.setInitialParameters(new RandomVariable[] { 0, 0 });
77    * optimizer.setWeights(new RandomVariable[] { 1, 1 });
78    * optimizer.setMaxIteration(100);
79    * optimizer.setTargetValues(new RandomVariable[] { 5, 10 });
80    *
81    * optimizer.run();
82    *
83    * RandomVariable[] bestParameters = optimizer.getBestFitParameters();
84    * </code>
85    * </pre>
86    *
87    * See the example in the main method below.
88    *
89    * <p>
90    * The class can be initialized to use a multi-threaded valuation. If initialized
91    * this way the implementation of <code>setValues</code> must be thread-safe.
92    * The solver will evaluate the gradient of the value vector in parallel, i.e.,
93    * use as many threads as the number of parameters.
94    * </p>
95    *
96    * Note: Iteration steps will be logged (java.util.logging) with LogLevel.FINE
97    *
98    * @author Christian Fries
99    * @version 1.6
100    */
