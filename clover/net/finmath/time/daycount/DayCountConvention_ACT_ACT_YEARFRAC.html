14    * Implementation of ACT/ACT as in Excel (2013).

16    * Calculates the day count by calculating the actual number of days between startDate and endDate.
17    *
18    * A fractional day is rounded to the approximately nearest day.
19    *
20    * The day count fraction is calculated using ACT_ACT_YEARFRAC convention, that is, it is calculates the daycount fraction
21    * corresponding to the Excel (2013) function YEARFRAC(startDate, endDate, 1). The day count fraction is calculated by dividing
22    * the actual number of days between start and end date by a denominator, where the denominator is determines as follows:
23    * <ul>
24    * <li>If the interval from start to end spans more than one year fraction, denominator is the average number of days per year</li>
25    * <li>If the interval from start to end spans less or equal one year and if start and end fall in a leap year, denominator is 366.0</li>
26    * <li>If the interval from start to end spans less or equal one year and if either falls in a leap year and if February 29th of that leap year lies in between start and end (or is equal to start or end), denominator is 366.0</li>
27    * <li>If the interval from start to end spans less or equal one year and neither of the above applied denominator is 365.0</li>
28    * </ul>
29    *
30    * <p>
31    * <strong>
32    * This method is not identical to ACT/ACT ISDA. For ACT/ACT ISDA use {@link net.finmath.time.daycount.DayCountConvention_ACT_ACT_ISDA}.
33    * </strong>
34    * </p>
35    *
36    * <p>
37    * In addition the method has the defect, that it is not additive (if you consider small intervals). For example
38    * YEARFRAC(30.12.2011, 04.01.2012, 1) is not equal to YEARFRAC(30.12.2011, 01.01.2012, 1) + YEARFRAC(01.01.2012, 04.01.2012, 1).
39    * </p>
