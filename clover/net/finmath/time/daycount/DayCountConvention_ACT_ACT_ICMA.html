16    /**
17    * Implementation of ACT/ACT ICMA.
18    *
19    * Calculates the day count by calculating the actual number of days between startDate and endDate.
20    *
21    * A fractional day is rounded to the approximately nearest day.
22    *
23    * The day count fraction is calculated using ACT_ACT_ICMA convention, that is, the
24    * day count fraction is <i>n/(f*m)</i>, where
25    * <ul>
26    * <li> <i>n</i> is the number of days between period start and the given day,</li>
27    * <li> <i>m</i> is the number of days between period start and period end,</li>
28    * <li> <i>f</i> is the number of periods within a year</li>
29    * </ul>
30    * where the start date is included in the counting and the end date is excluded in the counting.
31    *
32    * In contrast to other daycount conventions this daycount conventions requires information on an
33    * associated schedule of periods. It depends on the definition of period start and period end
34    * and on the frequency.
35    *
36    * This is implemented by providing an object implementing an <code>net.finmath.time.Period</code>
37    * upon construction.
38    *
39    * <p>
40    * The class can be constructed using isCountLastDayNotFirst = true, which will change its behavior compared to above.
41    * In this case the start date is excluded in the counting and the end date is included in the counting.
42    * This means that from 31.12.2014 to 01.01.2015 the day count fraction is 1/365 since 2015 is not a leap year.
43    * </p>
44    *
45    * Note: The daycount method {@link DayCountConvention_ACT_ACT_ISDA} is a special case of this method
46    * for frequency being one, having annual periods from January 1st to December 31st.
47    *
48    * @author Christian Fries
49    * @version 1.0
50    */
